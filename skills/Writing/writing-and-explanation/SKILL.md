# Writing and Explanation

Make writing clear, direct, useful, and appropriate for the audience.

## First decide the surface

- User-facing app copy: short, helpful, low-friction.
- Technical explanation: build from mental model to details.
- Proposal/spec: decision-oriented, tradeoffs explicit.
- Release note: what changed, why it matters, migration impact.

## Rules

- Use plain English.
- Avoid vague adjectives unless backed by facts.
- Prefer concrete nouns and verbs.
- Keep claims aligned with implementation.
- Remove filler, hype, and repetition.
- Define unfamiliar terms once.
- Use examples when they reduce ambiguity.

## Explanation structure

For technical explanations:

1. One-sentence answer.
2. Mental model.
3. Key moving parts.
4. Example or command.
5. Caveats.

## Rewrite workflow

1. Identify audience and goal.
2. Preserve intended meaning.
3. Remove ambiguity.
4. Improve flow and specificity.
5. Check for overclaiming.

## Output

Provide finished text first when the user wants copy they can paste. Add a short note only when useful.
